# AI-Assisted Worklog

Краткий журнал того, как ИИ использовался в процессе работы над решением.

В работе использовались два инструмента:

- Claude Code
- Codex

Оба использовались как инженерные ассистенты для анализа, диагностики и ускорения рутинных этапов, но итоговые решения проверялись на реальном локальном окружении.

## 1. Первичный аудит

- разобрана структура `frontend-challenge`
- сопоставлен текущий frontend с требованиями челленджа
- выбран backend fork `engineer-challenge` как источник реального auth-контракта
- на этом этапе использовались ИИ-подсказки для быстрого gap-analysis по требованиям

## 2. Архитектурная доработка frontend

- оформлена модульная структура `app / modules / shared`
- выделены базовые UI-компоненты
- добавлены server routes и session-oriented инфраструктурные helpers
- часть архитектурных вариантов обсуждалась с ИИ до внесения изменений в код

## 3. Интеграция с Ory Kratos

- проверены реальные browser flows Kratos
- исправлена работа через same-origin proxy
- устранены ошибки, связанные с cookies, CSRF и browser session
- учтён двухшаговый registration flow
- использовались ИИ-инструменты для ускорения диагностики контракта и поиска расхождений между UI и backend

## 4. Диагностика нестандартных ответов backend

- исследованы ошибки инициализации flow
- обработан сценарий, где первый шаг регистрации возвращает `400`, но flow должен продолжаться
- улучшен mapping backend-ошибок в пользовательские сообщения
- ключевые гипотезы проверялись не только текстово, но и через реальные запросы к локальному окружению

## 5. Надёжность и тесты

- добавлены и обновлены unit/integration тесты на критичные сценарии
- добавлен тест на двухшаговую регистрацию с промежуточным `400`
- подтверждена работоспособность сценариев через локальное окружение

## 6. Документация

- README переписан под реальные требования челленджа
- зафиксированы assumptions по backend-контракту
- описаны ключевые инженерные решения и trade-offs
- добавлена `.agents` директория для прозрачности использования ИИ

## Итог

ИИ использовался как ускоритель инженерной работы, но ключевые решения принимались после проверки кода, контракта backend и реального поведения локального окружения.
